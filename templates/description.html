Wstęp

Serwis został zrobiony na potrzeby zalicznia projektu na przedmiot "Architektura i Technologie Systemów internetowych". Prowadzony przez:

  • dr hab. inż. Bartłomiej Jacek Kubica
  • dr inż. Adam Kozakiewicz

Założenia postawine przez autora:

  • Stworzenie prostego serwisu typy portfolio zawierającego opis projektów autora
  • Do serwisu w łatwy sposób powwino dać się dodawać nowe projekty
  • Serwis powinien działać na urządzeniach mobilnych
  • Podtwarz tworzenia serwisu autor powinien zdobyć widze na temat tworzenia stron internetowych

Użyte technologie:

  • Python 3.9
  • Python web framework: Flask
  • Template language: JINJA2
  • HTML5
  • CSS3
  • Document Database: MongoDB

Wymagania postawione serwisowi przez prowadządzych:

  • Wykorzystanie co najmniej technologii HTML4. W projekcie użyto HTML5
  • Wykorzystanie "nietrywialnego" formatowania CSS. Została użyta techologia CSS3 oraz metodologia BEM(block-element-modifaier) aby foramtować HTML
  • Kompatybilność z popularnymi przeglądarkami internetowymi. Założenie spełnione. Strona działa poprawnie na Safari, Firefox, Chrome, Opera
  • łagodna degradacja, to znaczy serwis powinien działać na starszych przeglądarkach. ++++
  • Dostępność dla niewidomych. Każde zdjecie posiada dobrze je opisująca classe 'alt'. Serwis składa się z
  • Wykorzystanie innych technologii poza HTML i CSS. Zostały wykorzystane skrypty po stronie serwera napisane w jezyku Python z użyciem frameworku FLASK, oraz dane na temat projektów są przechowywane w bazie danych NoSQL MongoDB.
  • Serwis powinien zawierać dobrze przemyślany interfejs. Ocena tego punktu jest bardzo subiektywna ale według autora został on spełniony. Każda interaktywna część interfejsu w jakiś sposób zmiena sie po najechaniu na nią co pokazuje użytkownikowi, że może wejść w interakcjie z nią
