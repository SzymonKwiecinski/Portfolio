Wstęp

Serwis został zrobiony na potrzeby zaliczenia projektu na przedmiot "Architektura i Technologie Systemów internetowych". Prowadzony przez:

  • dr hab. inż. Bartłomiej Jacek Kubica
  • dr inż. Adam Kozakiewicz

Założenia postawione przez autora:

  • Stworzenie prostego serwisu typy portfolio zawierającego opis projektów autora
  • Do serwisu w łatwy sposób powinno dać się dodawać nowe projekty
  • Serwis powinien działać na urządzeniach mobilnych
  • Podczas tworzenia serwisu autor powinien zdobyć wiedzę na temat tworzenia stron internetowych

Użyte technologie:

  • Python 3.9
  • Python web framework: Flask
  • Template language: JINJA2
  • HTML5
  • CSS3
  • Document Database: MongoDB

Wymagania postawione serwisowi przez prowadzących:

  • Wykorzystanie co najmniej technologii HTML4. W projekcie użyto HTML5
  • Wykorzystanie "nietrywialnego" formatowania CSS. Została użyta technologia CSS3 oraz metodologia BEM(block-element-modifaier) aby sformatować HTML
  • Kompatybilność z popularnymi przeglądarkami internetowymi. Założenie spełnione. Strona działa poprawnie na Safari, Firefox, Chrome, Opera
  • Dostępność dla niewidomych. Każde zdjęcie posiada dobrze je opisującą classe 'alt'. Strona została zwalidowana na stronie "https://www.experte.com/accessibility" pod kątem dostępności dla niepełnosprawnych przechodząc testy na 95%
  • Wykorzystanie innych technologii poza HTML i CSS. Zostały wykorzystane skrypty po stronie serwera napisane w języku Python z użyciem frameworku FLASK, oraz dane na temat projektów są przechowywane w bazie danych NoSQL MongoDB.
  • Serwis powinien zawierać dobrze przemyślany interfejs. Ocena tego punktu jest bardzo subiektywna ale według autora został on spełniony. Każda interaktywna część interfejsu w jakiś sposób zmienia się po najechaniu na nią co pokazuje użytkownikowi, że może wejść z nią w interakcję
